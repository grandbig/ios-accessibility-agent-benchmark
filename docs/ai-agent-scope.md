# AI Agent のスコープ — ツリー型 vs ビジョン型

本ベンチマークが「AI Agent」と言うとき、どの種類の AI Agent を指しているのかを明確にする。
モバイルを操作する AI Agent には大きく2つのモダリティがあり、**得意・不得意（＝効く問題）が異なる**。

| | ツリー型（accessibility-tree） | ビジョン型（screenshot + LLM） |
| -- | -- | -- |
| 入力 | アクセシビリティツリー（role / label / value） | 画面のスクリーンショット（画素） |
| 操作 | 要素を role/label/id で選び操作 | 画素を見て座標をタップ |
| 例 | **agent-device（snapshot）**、XCUITest、Maestro | computer-use 系（スクショ＋座標）、本検証では「人＝視覚モデル」が代理 |
| 本ベンチの計測 | **直接計測した対象**（findings はすべてこれ） | 下記の対比デモで補完 |

> 重要: 本ベンチマークで実測した「Toggle 中央タップ／親 identifier の上書き／装飾UIは操作対象にならない」等の
> findings は、**ツリー型 AI Agent（agent-device）に対するもの**。ビジョン型は別モダリティで、別の長所・弱点を持つ。

## 実証: 同じ要素でも、ツリー型は不可・ビジョン型は可

a11y を与えていない Canvas 描画ボタン（`deco.canvas`）で対比した（`docs/decorative.md` の①）。

**ツリー型の見え方**（agent-device snapshot）:
```
@e14 [other] "deco.canvas"      ← [button] ではない。ボタンとして選べない＝操作できない
```

**ビジョン型の操作**（スクリーンショットで「①の送信ボタン」を視覚的に特定 → 座標タップ）:
```
agent-device click 76 236       → 最後にタップ: canvas   ← 操作できた
```

→ **ツリー型は「ボタンとして認識されない」ので操作不能だが、ビジョン型は見た目で座標を叩けるので操作できる。**
装飾UI/カスタム描画に対しては、ビジョン型の方が「とりあえず押せる」ことがある。

## ただしビジョン型には固有の弱点

- **座標が脆い**: 実際にこのデモでも、最初にスクショから目分量で出した座標 `(45,118)` は外れ（反応せず）、
  要素の実フレーム中心 `(76,236)` で初めて当たった。スクロール位置・画面サイズ・レイアウト変更・描画の差で
  簡単にズレる。再現性が低い。
- **意味が分からない**: 画素からは「これは送信ボタン」と確信できない（OCR 頼み）。`deco.canvas` と
  `deco.canvasFixed` のように**見た目が同一の要素を意味で区別できない**。Switch の ON/OFF など
  状態値も視覚推定になり不確実。
- **コスト/速度**: 1 ステップごとに画像を解釈するため遅く・高コストになりがち。

一方ツリー型は **role/label/value で意味的に・安定的に・高速に**操作できるが、**アクセシビリティツリーに
出ない要素（Canvas/Gesture のみ）には盲目**になる。

## まとめ（スコープと含意）

- 本ベンチマークの AI 関連 findings は **ツリー型 AI Agent を直接計測**したもの。ツリー型は VoiceOver・
  XCUITest・Maestro と同じ「アクセシビリティツリー依存」の仲間で、結論が一致する。
- **ビジョン型は別モダリティ**。装飾UI のように「ツリーに出ない要素」を座標で押せる利点があるが、
  座標の脆さ・意味理解の弱さ・コストという弱点がある。**「ビジョン型なら何でも押せるから設計は不問」ではない**。
- 設計の含意は変わらない：**意味のある label と正しい role(trait) を与える**ことで、ツリー型 AI Agent・
  テスト・VoiceOver のすべてに効く。ビジョン型に対しても「視覚的に明快で、テキストが OCR 可能」な
  UI の方が安定する。どちらのモダリティにも、結局**意味情報を持った素直な UI**が強い。

（注: ビジョン型の対比は、利用可能なビジョン専用エージェントがないため「スクリーンショットを視覚的に解釈して
座標を決め、agent-device の座標タップで実行する」形で代理実証した。専用のビジョン型エージェントでの追検証は
今後の課題。）
